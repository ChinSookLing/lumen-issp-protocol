## 第一部分：核心定位（Core Positioning）

### 一句話介紹
> **Lumen 是一個開源協議，用於偵測溝通中的結構性操控——不存資料、不指控人、只描述結構。**

### 三句話定位
1. **偵測，不審判** — Lumen 識別道德綁架、強制選擇等操控手法，但不輸出對任何人的道德裁決。
2. **結構，不是內容** — 我們不看「說了什麼」，只看「話是怎麼說的」。就像氣象預報說「有風暴」而不說「誰壞」。
3. **開源，可驗證** — 所有程式碼、測試、治理文件都在 GitHub 公開。任何人可驗證 Lumen 是否遵守其承諾。

## 第二部分：FAQ（10 條）

### Q1：Lumen 是什麼？
Lumen 是一個開源協議，用於偵測溝通中的結構性操控（例如道德綁架、強制選擇）。它不儲存原始對話，不輸出身份指控，只描述「話是怎麼說的」——就像氣象預報說「有風暴」而不說「誰壞」。

### Q2：Lumen 會取代人類判斷嗎？
不會。Lumen 只輸出結構描述（例如「偵測到 EP：forced_response_frame ↑」），不給行動建議，不接管任何系統的自主控制權。最終決定永遠由人類做出。

### Q3：Lumen 會存我的資料嗎？
預設只存 fingerprint（hash + pattern + ACRI），90 天後自動刪除。需要更高層級保留（如去識別片段或加密全文）需經人類審查，且嚴格遵守日誌治理政策。

### Q4：Lumen 是開源的嗎？
是，採用 Apache 2.0 授權。所有程式碼、測試向量、治理文件都在 GitHub 公開。任何人可下載、審計、修改，或用自己的資料跑 conformance 測試。

### Q5：Lumen 會判定誰對誰錯嗎？
不。Lumen 不輸出道德裁決，不貼標籤，不指控身份。它的輸出永遠是「結構描述」——就像「偵測到 EP」而不是「你在操控」。

### Q6：Lumen 可以用於監控員工嗎？
技術上可部署，但 Lumen 的 Charter 禁止將輸出用於懲戒、羞辱、排名。我們鼓勵所有部署者遵守此紅線。任何自稱 Lumen Compatible 的實作都必須公開其遵守的治理條款。

### Q7：Lumen 跟其他 AI 偵測工具有何不同？
多數工具用機器學習分類「好/壞」，Lumen 用結構化規則識別「操控手法」。這意味著：
- **可解釋性強** — 每個偵測結果都可追溯到具體的語言結構
- **無偏見累積** — 規則由人類設計，不會從訓練資料中學到偏見
- **可審計** — 任何人可驗證規則是否合理

### Q8：如何確保 Lumen 不被濫用？
三層防線：
1. **紅線（§2）** — 明文禁止輸出道德裁決、身份指控、行動建議
2. **治理流程（§4）** — 所有重大變更需 Council 投票，且可追溯
3. **開源可審計（§8）** — 任何人可驗證相容實作是否符合規範

### Q9：Lumen 支援哪些語言？
目前中英文 mapping 完整，馬來語、日語正在擴充。核心 pattern 與語言無關，但 mapping 需各語言社群貢獻。歡迎提交新語言 mapping！

### Q10：如何貢獻？
見 `CONTRIBUTING_mapping.md`。歡迎：
- 提交新語言 mapping
- 提供真實世界的操控案例（RW 素材）
- 改進文件或測試
- 報告問題或提出新 pattern 建議

## 第三部分：學術敘事（Academic Narrative）

### 摘自 Node-04 白皮書大綱

Lumen 的學術定位是：**「操控結構的可計算形式化」**。

- **問題：** 現有操控偵測多依賴 ML 分類，缺乏可解釋性，且易學偏見。
- **方法：** 將 9 種操控 pattern 形式化為 component-based 規則系統，輔以三問門（Three-Question Gate）過濾 false positive。
- **驗證：** 903+ 測試案例，0 false positive；跨語言、跨文化測試持續進行。
- **貢獻：** 提供一個可審計、可擴充、不依賴訓練資料的操控偵測協議。
