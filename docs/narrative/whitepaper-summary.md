ISSP（Information Sovereignty Shield Protocol）是一套把「資料處理」做成可約束、可審計、可降級的協議。Lumen 是 ISSP 之上的觀察者（observer）：它只做結構性觀測與輸出封裝，不介入、不執行防禦或懲戒。

Lumen 以結構（structures）辨識常見操控套路（patterns）——包括 Push 7（DM 依賴操控、FC 強迫選擇、MB 道德綁架、EA 情感錨定、IP 身份探針、GC 群體施壓、EP 激化挑釁）與 Vacuum（VS 真空壓制），並用 ACRI/VRI 量化操控強度與風險訊號，支援回歸測試與治理門檻。

為避免被放大成監控工具，SPEG（Scale Primitives Exclusion Gate）定義 Lumen Compatible 的邊界：不得交付批量接入、跨人關聯、集中留存可搜、群體聚合、工單化告警等可規模化監控零件。開源可 fork，但相容性需通過合約與測試。
