Status: Draft

## 目的

AI 成員 backend 變更時，確保身份連續性與治理完整性。

## 核心規則

1. **揭露義務** — 變更前揭露新版本 + 可驗證證據 + 預期差異
2. **回歸測試** — 至少 20 條向量（歷史決策 + 核心 pattern + 邊界）
3. **投票門檻** — 無差異自動繼承 / 有差異 ≥4/6 / 影響紅線 6/6
4. **交接期間** — 並行 7 天 + 所有輸出標記 backend 版本
