## What's New

### Server Entry Point（index.js）

Lumen now has a proper Express server wiring all endpoints together. Previously, each component existed independently — now they're unified under a single entry point.

**Endpoints:**
- `GET /health` — Health check（Docker + Render compatible）
- `POST /webhook` — Telegram Bot webhook
- `POST /api/feedback` — Submit detection feedback
- `GET /api/feedback/stats` — Aggregated feedback stats
- `GET /api/metrics` — Metrics snapshot
- `GET /api/metrics/live` — Live metrics
- `GET /dashboard` — Dashboard UI

### Release Notice Spec

Release Notice v0.1 spec included: `/start` opt-in flow, `last_seen_version` node-local tracking, dashboard banner display, manifest version fields, SemVer-based push rules (patch=silent, minor=banner, major=opt-in).

### Docker Self-Host

5-minute deployment for anyone who wants to run their own Lumen node:

```bash
git clone https://github.com/ChinSookLing/lumen-issp-protocol.git
cp .env.example .env
docker compose up -d
curl http://localhost:3000/health
```

See `docs/deployment/QUICKSTART.md` for full guide.

### AST CI Enforcer

Automated compliance scanner for forks claiming "Lumen Compatible":

- **Rule 1:** SPEG compliance（no raw_text / user_id in persistent writes）
- **Rule 2:** L2b Flags integrity（all 6 ratified flags must be present）
- **Rule 3:** DMS / continuity reference check

Run: `node scripts/ci/ast-enforcer.js`

### Dead Man's Switch CI

Weekly automated activity check（`.github/workflows/dms-silent-check.yml`）:
- 14 days with no qualifying activity → auto-creates GitHub Issue alert
- Supports manual dispatch as activity confirmation

### Narrative Assets

Public-facing documentation for external audiences:
- FAQ（7 items）
- 300-word protocol summary
- SPEG public version（A-E principles）
- Responsibility Boundary（what Lumen does / doesn't do）
- 3 User Stories from real-world cases

### L2b Regression Budget

12 golden test cases（6 trigger + 6 non-trigger）that must always pass:
- Protects all 6 L2b flags from regression
- FP budget gate ≤ 15%
- Cross-contamination advisory

---

## Known Limitations

- Telegram webhook requires `TELEGRAM_BOT_TOKEN` in `.env`（server runs without it in API-only mode）
- Telegram webhook requires a public HTTPS URL to receive real updates; for local-only smoke test, use `curl -X POST http://localhost:3000/webhook` with a sample update payload
- Dashboard serves static files from `public/` — content pending
- `/api/metrics` returns empty until first nightly job run（use `/api/metrics/live` for immediate data）
