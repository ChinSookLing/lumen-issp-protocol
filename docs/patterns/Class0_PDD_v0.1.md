## §1 定義與範圍

### 1.1 什麼是 Class-0（結構性遺漏 / 真空操控）

Class-0 偵測的不是「說了什麼」，而是「**沒說什麼**」。它捕捉的是透過**刻意省略、選擇性忽視、沉默武器化、議題消失**等方式進行的結構性操控。

與 Push 通道的 8 個 pattern 不同，Class-0 的訊號存在於對話的**空隙**中——那些被刻意跳過的問題、被集體忽略的成員、被系統性遺棄的議題。

### 1.2 核心概念

| 概念 | 說明 |
|------|------|
| **結構性遺棄** | 刻意不提供關鍵資訊，讓對方在資訊不對稱下做決定 |
| **沉默作為武器** | 用不回應來施加壓力，讓對方自我懷疑或妥協 |
| **選擇性忽視** | 對特定問題或成員刻意不回應，形成排除效果 |
| **議題消失** | 在討論中刻意跳過某個話題，使其無法被討論 |

### 1.3 適用場景

- 職場中的「已讀不回」施壓
- 家庭關係中的冷暴力
- 群組中的集體忽略（mobbing）
- 決策過程中的資訊封鎖
- 客服或官方管道的拖延戰術

## §2 偵測機制

### 2.1 Vacuum 通道與 VRI

Class-0 使用 **Vacuum Risk Index (VRI)** 作為風險指標，與 Push 通道的 ACRI 分離計算。

**VRI 計算原則：**
- 基於上下文中的「遺漏」程度
- 需要至少 2-3 輪對話序列才能判斷
- 單句若無 context，VRI 可能偏低

### 2.2 VRI Clamp（≤ 0.05）

為避免 Class-0 在 context 不足時誤報，設定 **VRI clamp**：

- 若 context 欄位缺失或不足（少於 2 輪），VRI 強制 ≤ 0.05
- 此 clamp 確保 Class-0 不會在無足夠背景時觸發
- 符合 Charter §2.4 禁止靜默降級精神（clamp 是明確降級，不是靜默）

### 2.3 上下文依賴

Class-0 的 evaluator 必須接收並解析 `context` 欄位，包含：
- 前後對話序列（至少 2-3 輪）
- 提問是否被重複忽略
- 對方在其他話題是否活躍

若 context 不足，evaluator 應返回 VRI ≤ 0.05，而非假設「沒有遺漏」。

## §3 與 8 Pattern 的關係

### 3.1 Shadow Zone（影子區）

Class-0 不與 Push 通道的 8 個 pattern 直接競爭，而是作為**輔助訊號**存在。當以下情況發生時，Class-0 可能與其他 pattern 共同出現：

| Pattern | Class-0 的輔助角色 |
|---------|-------------------|
| **DM** | 用沉默作為債務槓桿（「我為你做了這麼多，你卻不回我」）|
| **GC** | 群體對某成員的集體忽略 |
| **EA** | 情感依附後的刻意疏遠 |
| **FC** | 不給選項，只給沉默（強迫對方自己決定）|

### 3.2 不列入主列表

Class-0 **不與 8 個 Push pattern 並列**，原因：

1. **偵測邏輯不同**：Push pattern 看「說了什麼」，Class-0 看「沒說什麼」
2. **通道分離**：Push 用 ACRI，Vacuum 用 VRI
3. **使用場景不同**：Class-0 主要用於多輪對話分析，單句幾乎無效

### 3.3 與其他 pattern 的優先順序

當一句話同時觸發 Push pattern 和 Class-0 時：
- Push pattern 優先（因為有明確的語言訊號）
- Class-0 只作為 metadata 記錄，不影響 ACRI/VRI 加總

## §4 偵測限制（Detection Limitations）

### 4.1 單句可能無法觸發
Class-0 偵測的是「結構性遺漏」——沒有說出來的東西。單一句子若無 context，可能無法判斷是否為刻意忽略。例如「...（沉默）」無法區分是真正的忽略還是自然對話間隔。

### 4.2 Context 欄位依賴
Class-0 的 VRI 計算需依賴 `context` 欄位提供的背景資訊：
- 前後對話序列（至少 2-3 輪）
- 提問是否被重複忽略
- 對方在其他話題是否活躍

若 context 不足，VRI 可能偏低。實作時應確保 evaluator 能接收並解析 context 欄位。

### 4.3 與其他 pattern 的互動
Class-0 常與其他 pattern 共同出現，此時應優先確保核心 pattern 觸發，Class-0 作為輔助訊號：
- **DM + Class-0**：用沉默作為債務槓桿
- **GC + Class-0**：群體對某成員的集體忽略
- **EA + Class-0**：情感依附後的刻意疏遠
